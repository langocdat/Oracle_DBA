# Tổng quan
```
Trong Oracle Database, Materialized View (MView) (hay còn gọi là View hiển thị vật lý) là một đối tượng cơ sở dữ liệu chứa kết quả của một câu lệnh truy vấn (Query).
Khác với View thông thường (chỉ là một câu lệnh SQL được lưu lại và chạy lại từ đầu mỗi khi bạn gọi),
Materialized View thực sự lưu trữ dữ liệu vật lý trên đĩa cứng giống như một bảng thông thường.

Cơ chế làm mới:
- FAST Refresh: Chỉ cập nhật những thay đổi dựa trên một nhật ký thay đổi gọi là Materialized View Log được tạo ở bảng gốc.
- Complete Refresh: Oracle sẽ chạy lại hoàn toàn câu lệnh truy vấn ban đầu và nạp lại toàn bộ dữ liệu vào MView
- FORCE Refresh: Oracle sẽ cố gắng chạy FAST refresh trước; nếu không khả thi (ví dụ: không có MView Log), nó sẽ tự động chuyển sang COMPLETE refresh.

Thời điểm làm mới (Refresh Times)
- ON COMMIT: Dữ liệu trong MView tự động cập nhật ngay khi có giao dịch (Transaction) được COMMIT thành công ở các bảng gốc.
- ON DEMAND: Dữ liệu chỉ được làm mới khi người dùng hoặc hệ thống chủ động gọi các hàm/thủ tục (ví dụ: DBMS_MVIEW.REFRESH).
- START WITH ... NEXT ...: Thiết lập lịch trình (Scheduler) để tự động làm mới sau một khoảng thời gian định sẵn (ví dụ: mỗi tối lúc 12h).
```

	-- Tạo MVIEW
CREATE MATERIALIZED VIEW mv_doanh_thu_thang
BUILD IMMEDIATE -- Nạp dữ liệu ngay khi tạo
REFRESH FORCE -- Ưu tiên Fast Refresh, nếu không được thì Complete
ON DEMAND -- Làm mới khi có yêu cầu hoặc theo lịch
START WITH SYSDATE -- Bắt đầu ngay bây giờ
NEXT SYSDATE + 1 -- Tự động làm mới sau mỗi 1 ngày
AS
SELECT ma_cua_hang, SUM(doanh_thu) AS tong_doanh_thu
FROM ban_hang
GROUP BY ma_cua_hang;

-- drop MVIEW 
DROP MATERIALIZED VIEW TEST;
CREATE MATERIALIZED VIEW TEST
BUILD IMMEDIATE
REFRESH COMPLETE
ON DEMAND
START WITH SYSDATE
NEXT SYSDATE + (1/1440)
AS
SELECT COUNT(*) FROM employee;

-- Check data trong MVIEW
SELECT * FROM TEST;

ALTER MATERIALIZED VIEW TEST REFRESH ON DEMAND;

BEGIN
  DBMS_SCHEDULER.CREATE_JOB (
    job_name        => 'JOB_REFRESH_MVIEWS_NODE2',
    job_type        => 'PLSQL_BLOCK',
    -- Khối lệnh PL/SQL thực hiện refresh danh sách MView
    job_action      => 'BEGIN 
                          DBMS_MVIEW.REFRESH(''MV1'', ''F''); 
                          DBMS_MVIEW.REFRESH(''MV2'', ''F''); 
                          DBMS_MVIEW.REFRESH(''MV3'', ''F''); 
                        END;',
    start_date      => SYSDATE,
    -- Ví dụ: Lịch trình chạy mỗi tiếng một lần
    repeat_interval => 'FREQ=HOURLY; INTERVAL=1', 
    job_class       => 'CLASS_NODE2', -- ĐÂY LÀ ĐIỂM QUYẾT ĐỊNH: Ép job chạy trên node 2
    enabled         => TRUE,
    comments        => 'Job tu dong refresh cac MView duy nhat tren Node 2'
  );
END;
/
